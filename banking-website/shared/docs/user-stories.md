# User stories (preview)

Beginner-friendly scenarios that explain who needs what and why the UI/API exist.

1. **Register for an account (customer):** As a prospective customer, I want to sign up with my email and password so I can access my dashboard.
2. **Log in and see balances (customer):** As a returning customer, I want to see my current balances and recent transactions immediately so I know my cash position.
3. **Move money with confidence (customer):** As a customer, I want to transfer funds to a saved payee and receive a clear receipt so I can confirm the money went to the right place.
4. **Manage my profile (customer):** As a customer, I want to update contact info and security settings so support can reach me and my account stays safe.
5. **Stay informed (customer):** As a customer, I want to turn on notifications for logins, transfers, and large withdrawals so I can catch suspicious activity quickly.
6. **Recover access (customer):** As a locked-out customer, I want to reset my password and verify via email so I can regain access without contacting support.
7. **Monitor accounts (admin):** As an operations admin, I want to view and filter flagged transactions so I can spot fraud patterns.
8. **Assist users (admin):** As an admin, I want to reset a userâ€™s password or disable an account when security issues arise.
9. **Audit for compliance (admin):** As a compliance officer, I want to export or review an audit trail of admin actions so we meet regulatory requirements.
10. **Provide human help (support officer):** As a support officer, I want my contact info available via the `officer-contact.json` file so customers and admins can reach me when automated flows are not enough.
